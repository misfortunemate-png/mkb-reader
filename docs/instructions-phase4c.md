## 実装順序

1. **rewriteEngine.js** — importedAssetsの適用ロジック追加。insertedAssetsと同じ方式で、ImportedAsset型のデータをMD内に画像タグとして挿入する。data-asset-id属性を付与すること
2. **useLibrary.js** — 結合ノード作成関数の追加。ファイル接続のデータモデルをここで決定する
3. **LibraryEditor.jsx** — ファイル接続UI。編集モードで複数アイテムを選択→「結合」で新規LibraryNodeを作成
4. **ContextMenu.jsx** — 画像長押しメニューに「ライブラリに切り出す」を追加
5. **App.jsx** — 画像切り出しコールバックの接続。ライブラリ一覧→フォルダ選択→ノード選択のフロー
6. **LibraryView.jsx** — 編集モードに「結合」ボタンと「MKBとして出力」ボタンを追加
7. **LibraryExport.jsx** — ツリー走査→BookEntry取得→rewrite適用→edits適用→フォルダ→チャプター構造マッピング→JSZip→.mkbダウンロード
8. **結合テスト** — 結合ノードの削除で元BookEntryが消えないことを確認

## 完了条件

### 29.1 ファイル接続
- [ ] 編集モードで複数アイテムを選択し「結合」で新規ノードが作成される
- [ ] 結合されたノードをビューアで開くと、各ファイルの内容が自然なチャプター構造で表示される
- [ ] 各元ファイルの読み替えルール（層Aのrewrite）が適用された状態で結合される
- [ ] 結合ノードを削除しても元のBookEntryは消えない

### 29.2 画像切り出し
- [ ] ビューアで画像を長押し→コンテキストメニューに「ライブラリに切り出す」が表示される
- [ ] ライブラリ→フォルダ→ノード選択のフローで挿入先を指定できる
- [ ] 切り出した画像がImportedAssetとして対象ノードのeditsに保存される
- [ ] 対象ノードをビューアで開くと切り出した画像が表示される

### 29.3 mkbエクスポート
- [ ] 編集モードで「MKBとして出力」ボタンが表示される
- [ ] ライブラリまたはフォルダ単位でmkbファイルがダウンロードされる
- [ ] ダウンロードしたmkbをmkb-readerで再度開き、内容が正しいことを確認できる
- [ ] フォルダ構造がmkbのチャプター構造に正しくマッピングされている
- [ ] importedAssetsの画像がassets/に収集されている

### 回帰確認

- [ ] 既存のライブラリ機能（§28: ドリルダウン・フォルダ・アイテム追加・編集モード）に回帰がない
