## 1. 目的

自分にとって大切なテキストを、美しいタイポグラフィで読み返す場所の原型を作る。

この段階で確立するもの：
- フォント・余白・配色のデザイン言語（後続プロダクトB・Cの基準）
- MarkBook（.mkb）フォーマットへの対応（日本語圏初のmkbリーダー）
- モバイルでの読書体験の基本形

## 2. ベンチマーク

### Markant（macOS、Lakoy ApS、デンマーク、2026年）
- 「Markdownはすべてのエディタが書く。だが美しく読む場がない」という問題意識
- MarkBook（.mkb）フォーマットを提唱：ZIP内にindex.md + assets + metadata
- CC0ライセンスのオープン仕様、「開発者が一日でリーダーを作れる」設計原則
- 4テーマ（System / Paper / Ink / Academic）、コンテンツ幅3段階
- **持っていないもの：** ページネーション、縦書き、日本語フォント選択、モバイル対応

### 本プロダクトの差別化
- 日本語タイポグラフィ（明朝体選択、将来の縦書き）
- モバイルファースト（PWA、Androidファイルピッカー）
- ページネーション（Markdownビューアとして世界初の試み）
- 「自分の営みを鑑賞する場」という個人的目的

## 3. 入力フォーマット

| フォーマット | 扱い | v0.1対応 |
|---|---|---|
| .mkb | 第一級。ZIP展開→index.md + pages/ + assets/を描画 | ○ |
| .md | 「index.mdだけのMarkBook」と同等に扱う | ○ |
| .txt | プレーンテキストとして描画（Markdown解釈なし） | ○ |
| .html | 将来対応 | × |
| PDF | 将来対応 | × |
| チャットログJSON | 将来対応（.mkbへの変換パイプライン） | × |

### ファイル読み込み方式
- **主：** ファイル選択ボタン（`<input type="file" accept=".mkb,.md,.txt">`）
  - モバイル：Androidファイルピッカーが起動。端末内ファイルおよびGoogle Driveから直接選択可能（API連携不要）
  - デスクトップ：同じボタンでOS標準のファイル選択ダイアログ
- **補：** ドラッグ＆ドロップ（デスクトップのみ、優先度低）

## 4. 機能要件

### §1: MarkBookファイルの読み込みと展開
- .mkbファイル（ZIP）をブラウザ上で展開する
- index.mdをエントリポイントとして描画する
- pages/ディレクトリ内のMDファイルをチャプターとして認識する
- assets/ディレクトリ内の画像をMD内の参照に基づき表示する
- markbook.yamlがあればタイトル・著者等のメタデータを表示する
- 単体.mdファイルは「index.mdのみのMarkBook」として統一処理する

### §2: Markdownパースと描画
- CommonMark準拠のMarkdownパース
- 見出し（h1〜h6）、段落、リスト、引用、コードブロック、テーブル、画像の描画
- GFM拡張（タスクリスト、取り消し線、テーブル）
- wikilinks（`[[ページ名]]`）によるチャプター間リンク

### §3: チャプターナビゲーション
- mkb内の複数チャプターをサイドバーまたはドロワーで一覧表示する
- 現在のチャプターをハイライトする
- チャプター選択で描画内容を切り替える
- モバイルではハンバーガーメニューからドロワー表示

### §4: ページネーション
- CSS multi-columnによるビューポート単位のページ分割
- タップ（モバイル）またはクリック（デスクトップ）でページ送り
- 現在のページ番号 / 総ページ数の表示
- スワイプによるページ送り（モバイル）
- **フォールバック：** 連続スクロールモードへの切替を設定から可能にする

**【検証デプロイポイント】** §1〜§4の実装完了時点でGitHub Pagesにデプロイし、Pixel 10実機で以下を検証する：
- ページネーションの読書体験は「鑑賞」の名に値するか
- 日本語テキストの禁則処理・見出し切れ等の不具合の程度
- スワイプ/タップの操作感
- 検証結果により、§5以降の実装に進むか、§4の改善を優先するか判断する

### §5: 日本語フォント選択
- 以下の3フォントから選択（Google Fonts読み込み）
  - Noto Serif JP（標準的な明朝体）
  - Shippori Mincho（文学的な明朝体）
  - Zen Old Mincho（古典的な明朝体）
- 選択されたフォントのみ遅延ロードする（初回表示を軽くする）
- `font-display: swap`でFOUT対策
- 欧文フォント1種をペアリング（未定、仕様書段階で選定）

### §6: テーマ切替
- 3テーマ：ライト / ダーク / セピア
- CSS変数で配色を管理
- テーマ切替は即座に反映（ページリロード不要）

### §7: 行間・余白カスタマイズ
- プリセット3段階：ゆったり / 標準 / コンパクト
- line-height、padding、max-widthを連動して調整
- 変更は即座に反映

### §8: 設定保存
- フォント選択、テーマ、行間・余白、ページネーション/スクロール切替をlocalStorageに保存
- 次回起動時に復元

### §9: PWA基本対応
- マニフェスト設定（ホーム画面追加、アプリ名、アイコン）
- アプリシェルのキャッシュ（Service Worker）
- **含めない：** 開いたファイルのオフライン保持（v0.2以降）

## 5. v0.1に含めないもの（意図的な除外）

- 縦書き表示（v0.2以降）
- HTML / PDF対応
- チャットログ変換・取り込み
- プロダクトBからのエクスポート閲覧
- ブックマーク・読書進捗の記録
- 引用コピー・注釈ペースト
- 複数ファイルの本棚管理（IndexedDB）
- Google Drive API連携（ファイルピッカー経由で実質的にアクセス可能）
- 開いたファイルのオフラインキャッシュ
- markbook.yamlの高度な機能（署名、バージョン履歴等）
- 欧文専用フォントの追加選択肢

## 6. 技術選定（暫定）

| 技術 | 用途 | 理由 |
|---|---|---|
| React + Vite | フレームワーク | 既存アプリと統一 |
| react-markdown + remark/rehype | MDパース | 枯れた技術、プラグイン豊富 |
| JSZip | mkb展開 | ブラウザ上でのZIP展開の定番 |
| CSS multi-column | ページネーション | 業界標準（Bibi, epub.js, Readest全てが採用） |
| Google Fonts API | 日本語フォント | 無料、サブセット配信あり |
| vite-plugin-pwa | PWA | Viteとの統合が容易 |
| Tailwind CSS | スタイリング | 既存アプリと統一。テーマ切替はCSS変数 |
| GitHub Pages | ホスティング | 既存アプリと統一 |

## 7. 技術リスク評価

| リスク | 重度 | 対策 |
|---|---|---|
| CSS multi-columnのページネーション調整 | **高** | 連続スクロールへのフォールバックを用意。v0.1では「概ね動く」を受入基準とし、禁則処理の細部はv0.2 |
| 日本語Webフォントの初回読み込み | 中 | 選択フォントのみ遅延ロード。`font-display: swap` |
| mkb（ZIP）のブラウザ上展開 | 低 | JSZipで枯れた技術 |
| Androidファイルピッカーのエンコーディング | 低 | UTF-8前提。文字化け時はエラー表示 |
| localStorageの容量制限 | 低 | v0.1では設定値のみ保存（数KB） |
| PWA Service Workerのキャッシュ制御 | 低 | アプリシェルのみ。ファイルキャッシュはv0.2以降 |

## 8. 非機能要件

- モバイル（Pixel 10 Chrome）での閲覧を主要ユースケースとする
- デスクトップでも使えるが、モバイルファーストで設計する
- GitHub Pagesで配信（Tailscale経由でフランからもアクセス可能）
- リポジトリは単独（モノレポにしない。マージはPhase 4で検討）

## 9. 受入基準

1. ファイル選択ボタンから.mkbファイルを開き、index.mdが描画される
2. mkb内の複数チャプターがナビゲーション表示され、切り替えできる
3. mkb内のassets/画像がMD内で正しく表示される
4. 単体.mdファイルを開き、同じ描画品質で表示される
5. ページネーションでページ送りができる（タップ/スワイプ）
6. ページネーション/スクロールの切替ができる
7. 3種のフォント切替が即座に反映される
8. 3種のテーマ切替が即座に反映される
9. 行間・余白の変更が即座に反映される
10. 設定がブラウザ再起動後も維持される
11. Pixel 10のChromeで快適に操作できる
12. GitHub Pagesにデプロイされている
13. PWAとしてホーム画面に追加できる
