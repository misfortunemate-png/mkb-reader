## 作業手順

### Step 1: §20 モバイル表示問題の診断

**この問題はコードの問題ではない可能性が高い。**

1. 仕様書§2のデバッグ手順に従い、リモートデバッグで実viewport値を測定する
2. 結果を `_STATUS.md` に記録する
3. 判定:
   - `innerWidth` が 1080 → viewport metaが無視されている → 原因を特定
   - `visualViewportScale` が ~0.38 → ChromeのサイトZoom → ユーザー操作で解消、コード変更不要
   - 正常値（412, 1.0）が出る → 別の原因を調査
4. 別ブラウザ（Firefox）での表示確認を行い、結果を記録する
5. 原因が特定できた場合、修正を実施する。特定できない場合はPMに報告する

**禁止: 対症療法（フォントサイズ縮小等）は行わない。viewport metaは変更しない。**

### Step 2: §21 コンテキストメニュー

1. MarkdownRenderer.jsxのカスタムコンポーネントに `data-source-line` 属性を追加する
   - react-markdownのAST nodeから `node.position?.start?.line` を取得
   - `<p>`, `<h1>`〜`<h6>`, `<blockquote>`, `<li>`, `<pre>` に付与
   - 位置情報がない場合は属性を省略（graceful degradation）
2. ContextMenu.jsx を新規作成する
   - 長押し検出（700ms、指移動10px以下）
   - position: fixedで長押し位置に表示
   - 画面端のはみ出し補正
   - メニュー外タップで閉じる
   - テーマCSS変数に従うスタイリング
3. Paginator.jsx / スクロールモードの両方で長押しイベントを検出する
   - タップ（ページ送り）との競合回避: 700ms未満はタップ扱い
   - スワイプとの競合回避: 指移動10px超はスワイプ扱い
4. メニュー項目の実装:
   - 「この行を非表示にする」: `data-source-line` から行番号取得 → useRewrite.jsで hiddenRanges に追加 → 再描画
   - 「テキストを読み替える」: インライン入力UI表示 → useRewrite.jsで replacements に追加 → 再描画
   - 「ここに画像を差し込む」: `data-source-line` から行番号取得 → 既存ImageInserter.jsxのロジックを呼び出す
   - 「読み替え設定を開く」: 既存RewritePanel表示
5. 制約の実装:
   - ImageViewer.jsx / HtmlRenderer.jsx / JsonRenderer.jsx ではコンテキストメニューを無効化
   - MD表示モード（MarkdownRenderer.jsx経由）でのみ有効

### Step 3: §22 チャットログ変換の品質改善

1. 話者名のスタイリングを改善する
   - MarkdownRenderer.jsxで `**human**` / `**assistant**`（または読み替え後の名前）を含む行を検出
   - CSSクラス `speaker-human` / `speaker-assistant` を付与
   - reader.cssに話者名のスタイル追加（背景色、左ボーダー、パディング）
   - 読み替え後の話者名にも同じスタイルが適用されるようにする
     - **方式:** rewriteEngine.jsのspeakerNames置換時に、`**human**` → `**ショウゴ**` だけでなく、話者名の行自体に識別マーカーを付与する方式を検討。ただしrewriteEngineは純粋関数であり、CSSクラスの付与はMarkdownRenderer.jsx側の責務
2. ChatImporter UIの改善
   - 会話リストにタイトル検索フィルタを追加（input + リアルタイムフィルタ）
   - 各会話の最初の2メッセージをプレビュー表示（折りたたみ）
   - 大量変換時のプログレス表示
3. `<details>` 折りたたみのCSS multi-column内動作を確認する
   - 問題があれば `break-inside: avoid` を追加

### Step 4: §23 UI磨き込み

1. ヘッダーのアイコン整理
   - モバイル幅（412px）でアイコンが収まるか確認
   - 収まらない場合、≡ と ⚙ を常時表示、他（✏ / 🔖 / ↓）をオーバーフローメニュー（⋮）内に配置
2. 本棚画面の改善
   - ファイルタイプに応じたアイコン（絵文字で可: 📖 / 📝 / 🖼 / 💬 等）
   - lastOpenedAtの相対表示（Intl.RelativeTimeFormat使用）
3. エラーハンドリング統一
   - 簡易トーストコンポーネントを作成（画面下部に3秒表示→フェードアウト）
   - ファイル読み込みエラー、ZIP展開エラー、JSON解析エラーで使用

### Step 5: デプロイと検証

1. `npm run build` → `git push origin main`
2. Pixel 10実機で全テスト項目を確認
3. `_STATUS.md` を更新

## 禁止事項

- `BookEntry.fileData`を変更する処理を書かない
- `rewriteEngine.js`に副作用を持たせない
- viewport metaを変更しない
- モバイル表示問題に対する対症療法を行わない
- 仕様外の設計判断が必要な場合は作業を停止し報告する

## 完了条件

1. モバイル表示問題の原因が特定され、可能な場合は修正されている
2. リモートデバッグの結果が `_STATUS.md` に記録されている
3. テキスト長押しでコンテキストメニューが表示される
4. メニューから行非表示が即座に動作する
5. メニューからテキスト読み替えが即座に動作する
6. メニューから画像差し込みが動作する
7. 通常タップ（ページ送り）とメニュー表示が競合しない
8. 話者名が視覚的に区別できるスタイルで表示される
9. ChatImporterで会話タイトルの検索フィルタが動作する
10. ヘッダーのアイコンがモバイル幅に収まる
11. エラー時にトースト通知が表示される
12. GitHub Pagesにデプロイされている
