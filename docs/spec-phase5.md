## 4. 機能仕様

### §31: 次チャプター自動送り

**何を:** チャプターの最終ページ（ページモード）または末尾（スクロールモード）に到達したとき、操作を続けると自動的に次のチャプターに遷移する。

**どのように:**

#### 31.1 ページモード

usePagination.jsの`next()`で現在のページが最終ページ（`page === total - 1`）の場合、次チャプターへの遷移コールバック `onChapterAdvance` を発火する。同様に`prev()`で先頭ページ（`page === 0`）の場合、前チャプターへの遷移コールバック `onChapterRetreat` を発火する。

Paginator.jsxに `onChapterAdvance` と `onChapterRetreat` プロップを追加。これをApp.jsxでchapters配列のインデックスを進める/戻す処理に接続する。

```javascript
// usePagination.js の next() 拡張イメージ
const next = useCallback(() => {
  setPage((p) => {
    if (p >= total - 1) {
      // 最終ページの先で呼ばれた → チャプター送り
      onAdvanceRef.current?.();
      return p;  // ページ番号は変えない（チャプター切替で再計算される）
    }
    return p + 1;
  });
}, [total]);
```

#### 31.2 スクロールモード

スクロールモードでは、スクロール位置が末尾に到達した状態でさらに下方向にスワイプ（またはスクロール操作）した場合に `onChapterAdvance` を発火する。

具体的には、`scrollTop + clientHeight >= scrollHeight - 1`（1pxの余裕）の状態で追加のスクロール操作を検出したとき。ブラウザのoverscroll挙動（バウンス）との競合を避けるため、`overscroll-behavior: none` を`.paginator-frame`に設定し、末尾到達後のtouchendイベントで判定する。

先頭への遡りも同様。`scrollTop === 0` の状態で上方向スワイプで `onChapterRetreat` を発火。

#### 31.3 縦書きモード（§30）

縦書きモードはスクロールモード固定。スクロール方向が横（左方向）になるため、判定軸を `scrollLeft` / `scrollWidth` に切り替える。`writing-mode: vertical-rl` では `scrollLeft` が負の値をとるブラウザがあるため、`Math.abs(scrollLeft) + clientWidth >= scrollWidth - 1` で末尾判定する。

#### 31.4 遷移時の演出

チャプター遷移時、新しいチャプターの先頭（ページ0 / scrollTop=0）から表示を開始する。前チャプターへの遡り時は最終ページ / 末尾から表示する。lastPositionの保存は新しいチャプターに切り替わった時点で更新する。

チャプターがない場合（単一チャプターのmd/txtファイル）は何もしない。最終チャプターの末尾ではadvanceを発火しない。最初のチャプターの先頭ではretreatを発火しない。

**制約:**
- タップゾーン操作（§24）でも同様に末尾超えで次チャプターに進むこと
- スワイプ操作（§4）でも同様に動作すること
- キーボード操作でも同様に動作すること

### §32: 表紙画像

**何を:** BookEntryとLibraryNodeに表紙画像を持たせる。本棚・ライブラリの表示をリッチにする。

**どのように:**

#### 32.1 自動抽出

BookEntryの保存時（useBookshelf.jsのsaveBook）、fileTypeに応じて表紙画像を自動抽出する:

- `fileType === 'mkb'`: JSZipでfileDataを展開し、assets/内の最初の画像ファイルを取得
- `fileType === 'images'`（CBZ等）: 最初の画像ファイルを取得
- その他（md/txt/html/json/vertical）: 自動抽出なし

抽出した画像はuseImageResize.jsで幅200px程度のサムネイルにリサイズし、`BookEntry.coverImage` に保存する。

#### 32.2 手動設定

以下のいずれかで手動設定可能にする（UIの具体的なデザインはPG裁量）:

- Bookshelf.jsx: 本のメニューから「表紙を設定」→ 画像ファイル選択
- LibraryView.jsx: ノードのメニューから「表紙を設定」→ 画像ファイル選択
- ContextMenu.jsx: ビューア内の画像長押しから「表紙に設定」（任意実装）

手動設定もサムネイルサイズにリサイズしてから保存する。

#### 32.3 表紙画像がない場合の表示

表紙画像が未設定のアイテムはタイトル文字列をカード上に表示する（D-007）。デザインの詳細はPG裁量。参考として:
- fileTypeに応じた背景色の差異
- タイトルの文字サイズ・配置
- 著者名の副表示

#### 32.4 本棚での表示

Bookshelf.jsxの既存リスト表示に表紙サムネイルを追加する。表示方法（左端にサムネイル追加、グリッド表示への切替等）はPG裁量。

#### 32.5 IndexedDB

BookEntryのcoverImageはオプショナルフィールドとして追加。既存エントリは`undefined`のまま許容する。DB_VERSIONの変更は不要（フィールド追加のみ、インデックスなし）。

### §33: ライブラリ表示モード切り替え

**何を:** ライブラリのフォルダ内表示をカタログモード（グリッド＋表紙画像）とリストモード（コンパクト一覧）で切り替える。設定はフォルダ単位で記録する。

**どのように:**

#### 33.1 表示モード

**カタログモード（デフォルト）。** CSS Grid で表紙画像を大きく並べる。表紙がないアイテムはD-007に従いタイトル表示。フォルダはフォルダアイコン＋名前。タップでドリルダウンまたはビューア起動。

**リストモード。** 1行1アイテムのコンパクト表示。タイトル・著者・文字数を横並び。小さなサムネイル（あれば）。エクスプローラーの「詳細」表示に相当。

#### 33.2 切り替えUI

フォルダ内の画面上部（パンくずリストの近く）にアイコンボタンを2つ配置: グリッドアイコン（カタログ）、リストアイコン（リスト）。現在のモードをハイライト。

#### 33.3 フォルダ単位の記録

LibraryNode（type='folder'）に `viewMode: 'catalog' | 'list'` を追加。切り替え時にuseLibrary.jsで当該フォルダノードのviewModeを更新する。

トップレベル（rootNodes）のviewModeはLibraryオブジェクト自体に `rootViewMode?: 'catalog' | 'list'` を追加して保存する。

未設定の場合は `'catalog'` をデフォルトとする。

### §34: ライブラリごとインポート/エクスポート

**何を:** ライブラリ単位でデータを書き出し（エクスポート）・読み込み（インポート）する。別デバイスへの移行やバックアップに使う。

**どのように:**

#### 34.1 エクスポート

LibraryViewの編集モードまたはライブラリ選択バーのメニューから「ライブラリを書き出す」。

エクスポート対象:
- Library オブジェクト全体（nodes、rootNodes、name等）
- 各LibraryNode.edits（層Bの編集データ）
- 各ノードの coverImage
- 参照先BookEntryのfileData（エイリアスの場合でも実体を含める。別デバイスで復元できるようにするため）
- 参照先BookEntryのlocalSettings（display, rewrite, lastPosition）

形式はPG裁量（D-008）。

#### 34.2 インポート

LibraryViewの「+」メニューまたは設定から「ライブラリを読み込む」。

インポート時の処理:
- エクスポートファイルを読み込み、Libraryオブジェクトを復元
- 含まれるBookEntryのfileDataを本棚（層A）に復元。既に同じタイトル+同じ文字数のBookEntryが存在する場合は重複チェックを行い、既存のものを参照する（新規作成しない）
- LibraryNode.sourceBookId を復元先のBookEntry.idに付け替える

#### 34.3 制約

- エクスポートファイルのサイズは大きくなりうる（画像を含むmkbファイルが複数含まれるため）。進捗表示を出すこと
- インポート時に既存ライブラリと同名のものがあった場合のハンドリング（リネームまたは上書き確認）はPG判断
